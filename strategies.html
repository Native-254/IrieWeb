The strategy layer is deliberately boring in the best way: no broker calls, no account mutation, and no notification side effects. It just answers, "What does this model think should happen next?"

Why normalize signals?

A common signal enum lets the resolver compare very different strategies without caring how each one reached its conclusion. That also makes new strategies easier to add: implement the base interface, return the shared signal type, and let the resolver do portfolio-level arbitration.

class Strategy:
    def generate_signal(bars) -> Signal:
        return Signal.BUY | Signal.SELL | Signal.SHORT | Signal.COVER | Signal.HOLD
