Architecture

The bot is built as a set of focused modules: configuration, data, strategy generation, signal resolution, risk, execution, monitoring, and backtesting.

architecture breakdownv2.0

The design separates decisions from side effects. Strategy code can propose signals, risk code can size or reject them, and execution code can translate approved intent into broker orders.

LayerResponsibility
config/YAML settings, environment switches, symbol lists, risk limits, broker mode, and notification toggles.
data/Market data fetching, Parquet cache management, provider abstraction, and rate-limit behavior.
strategies/Independent strategy classes that calculate indicators and return normalized buy, sell, short, cover, or hold signals.
live/engine.pyThe orchestration loop that fetches data, runs strategies, resolves signals, checks risk, places orders, and reports state.
execution/Broker adapters, IBKR connectivity, paper/live modes, and order construction.
risk/Position sizing, exposure caps, heat checks, drawdown behavior, daily loss limits, and ATR stop logic.
monitoring/Discord, Telegram, health checks, and dashboard-facing status.
backtest/Historical simulation and performance analysis before a strategy reaches live operation.

Control Flow

data bars -> strategy signals -> resolver intent -> risk approval -> broker order -> notification -> dashboard state

This keeps failures contained. A data provider outage should not rewrite strategy rules; a broker adapter change should not alter risk math; a notification failure should not stop portfolio accounting.
