Data Pipeline

The data layer fetches bars, caches them for reuse, respects provider limits, and keeps the rest of the bot ready for new data sources.

The pipeline should make fresh data available without hammering providers or forcing strategy code to know where candles came from.

  • Fetching: retrieves intraday and historical bars from configured providers such as Yahoo Finance.
  • Caching: stores reusable datasets in compact files such as Parquet so repeated backtests and dashboard views stay quick.
  • Rate limiting: spaces requests, batches symbols, and falls back to cache when provider limits are near.
  • Provider flexibility: a common adapter interface makes it possible to add Polygon, Alpaca, IBKR historical data, or paid feeds later.
